# RPG_Ero — дизайн (v2)

**Дата:** 2026-06-26 (v2; v1 — 2026-06-25)
**Статус:** согласован, готов к плану реализации

## 1. Замысел

Текстовая браузерная одиночная RPG на правилах **Pathfinder 2e Remaster**, в духе MUD/RPG 80-х:
текстовый лог событий + командная строка + кнопки для частых действий, эстетика CRT-терминала.

- **Формат:** одиночная игра (single-player).
- **Тон:** фэнтези с взрослыми темами (18+), между вымышленными взрослыми персонажами. Взрослый контент — отдельный слой описаний, не зашитый в боевой движок.
- **Охват правил для v1:** ядро PF2e Remaster + **полное создание персонажа 1-го уровня**.
- **Источник данных правил:** **pf2.ru** (русский справочник PF2e) — парсинг и упаковка в собственную компактную схему.

## 2. Инфраструктура и согласование с home-page

Проект размещается на **той же ВМ** и **том же домене** (`telnor.ru`), что и репозиторий `home-page`, и
переиспользует его стек. Сетевые/доменные правки вносятся в репозиторий `home-page`, который рулит доменом
(создание базы, сети, поддомены в Traefik).

Окружение home-page (для справки):
- Docker Compose + **Traefik v3.6** (Let's Encrypt, маршрутизация по поддоменам, сети `web` внешняя + `internal`).
- **PostgreSQL 16** (контейнер `postgres`, том `pg_data`, сеть `internal`).
- Бэкенд: **FastAPI / Python 3.12**, SQLAlchemy 2.0 async + asyncpg, Alembic, pydantic-settings, JWT (python-jose) + bcrypt, **uv**, ruff, pytest-asyncio.
- Фронт: **React 19 + Vite + TS + Tailwind 4 + shadcn/ui**, TanStack Query, zustand, react-router, vitest; статика через nginx.

**Решения по инфраструктуре RPG_Ero:**
- **БД:** общий инстанс PostgreSQL 16, **отдельная база `rpg_ero`**. Бэкенд RPG_Ero подключается к
  существующему контейнеру `postgres` через внешнюю сеть `internal`. База `rpg_ero` создаётся отдельно
  (правка в `home-page`), home-page-приложение не затрагивается.
- **Поддомены:** `rpg.telnor.ru` (фронт) + `rpg-api.telnor.ru` (API), зеркально паттерну home-page.
- **Деплой:** собственный `docker-compose.yml` RPG_Ero, подключается к внешним сетям `web` и `internal`
  Traefik; маршрутизация по поддоменам через лейблы Traefik. Бэкенд-образ с `entrypoint.sh`,
  выполняющим `alembic upgrade head` перед стартом uvicorn — как в home-page.
- **Репозиторий / CI:** пока только локальный git. GitHub-проект и CI (**GitHub Actions**: lint/test/build образов)
  заводятся позже — это не блокирует начало разработки.

## 3. Архитектура

**Авторитетный бэкенд.** Движок правил и игровой цикл живут на сервере; браузер — «тонкий терминал».
Клиент общается с сервером через REST (аутентификация, сохранения, создание персонажа) и **WebSocket**
(живой поток игровых событий). Состояние игры — на сервере, защищено от подмены, и переиспользуемо при
будущем мультиплеере.

### Стек

| Слой | Технология |
|------|-----------|
| Бэкенд | Python 3.12 + FastAPI, SQLAlchemy 2.0 async + asyncpg |
| Движок правил | Чистый Python-пакет (без I/O), юнит-тесты pytest |
| Парсер данных | Playwright (headless) для обхода JS-защиты pf2.ru + парсинг HTML |
| Реалтайм | FastAPI WebSocket |
| БД / миграции | PostgreSQL 16 + Alembic |
| Менеджер пакетов | uv (бэк), npm (фронт) |
| Фронтенд | React 19 + Vite + TS + Tailwind 4 + shadcn/ui |
| UI | React-компонент «терминал» (CRT 80-х) + shadcn для конструктора персонажа |
| Линт / тест | ruff + pytest (бэк); ESLint + vitest (фронт) |
| Прокси / TLS | Traefik v3.6 (общий, по поддоменам) |

## 4. Структура репозитория

Полирепо по образцу home-page (backend / frontend / infra / docs):

```
RPG_Ero/
  backend/
    app/
      engine/        # чистый движок правил PF2e: броски, степени успеха, бой, персонаж
        chargen/     # чистая логика создания персонажа (повышения, владения, HP, фиты, снаряжение)
      game/          # игровой цикл, мир (локации), парсер команд, применение действий
      content/       # слой описаний/сцен (в т.ч. 18+) — данные, не логика
      api/           # REST-роутеры (auth, chargen, saves, health) + WebSocket-эндпоинт
      core/          # config (pydantic-settings), db, security (JWT), dependencies
      db/            # SQLAlchemy-модели, base
      schemas/       # Pydantic-схемы команд/событий/персонажа (контракт с клиентом)
      main.py
    alembic/         # миграции
    tools/           # скрипт парсинга pf2.ru (Playwright → наша схема)
    data/            # сконвертированные правила (JSON), фиксируем дату/версию снимка pf2.ru
    tests/
    Dockerfile, entrypoint.sh, pyproject.toml
  frontend/
    src/             # React-терминал, экраны конструктора персонажа, WS-клиент, состояние (zustand)
    Dockerfile, nginx.conf, package.json, vite.config.ts
  infra/
    docker/          # docker-compose.yml (внешние сети web/internal), лейблы Traefik
  docs/
  README.md          # включает атрибуцию pf2.ru + Paizo/ORC
```

**Принцип изоляции:** `engine` — чистые функции (состояние + действие → новое состояние + события),
без сети и БД, полностью тестируем. `engine/chargen` — самодостаточный под-движок создания персонажа.
`game` оркестрирует движок и мир. `content` (включая 18+) — отдельный слой данных, движок про него не знает.

## 5. Движок правил (ядро PF2e Remaster)

- **Бросок проверки:** `d20 + модификатор vs DC`; четыре степени успеха (крит-успех / успех / провал /
  крит-провал) с правилом ±10 и натуральными 20/1. Бросок инъектируется (детерминированный ГПСЧ) для тестов.
- **Характеристики:** 6 атрибутов и модификаторы, уровни владения (untrained → legendary), HP, AC,
  спасброски, DC класса/восприятия.
- **Бой на 3 действия:** инициатива, ходы, штраф за серию атак (MAP), урон и типы урона, небольшой набор
  состояний (например: раненый, без сознания).

## 6. Конструктор персонажа (Веха 1, ядро)

Создание персонажа **1-го уровня полностью по книге** PF2e Remaster. Самодостаточный модуль: своя логика,
данные, экраны и API — пригоден к использованию **отдельно от игры** (для НРИ).

**Логика (`engine/chargen`, чистые функции):**
- **Родословные + наследия (heritage)**, **предыстории (background)**, **классы**.
- **Повышения характеристик:** ключевая характеристика класса, повышения от родословной/предыстории/класса,
  4 свободных повышения; правило частичных повышений (apex-кэп ниже 18 на 1-м уровне); **добровольные
  изъяны** опционально.
- **Владения:** из класса (атаки, защита, спасброски, восприятие, DC класса), предыстории и навыков класса;
  обученные навыки (Int-бонус добавляет свободные).
- **Производные значения:** HP (родословная + класс + Con), AC, спасброски, восприятие, DC класса.
- **Стартовые фиты:** родовой фит, навыковый фит (где даёт предыстория), классовый фит (где даёт класс) —
  **с проверкой пререквизитов** и трейтов.
- **Снаряжение:** стартовое золото, базовый список снаряжения.
- **Валидация:** движок проверяет легальность всех выборов; UI авторитетно опирается на серверную валидацию.

**Выход:** лист персонажа + **экспорт в JSON** (печатный лист — позже).

**Уровни выше 1-го (level-up) — вне охвата v1.**

## 7. Конвейер данных (pf2.ru)

Скрипт в `backend/tools` парсит **pf2.ru** через Playwright (headless-браузер проходит JS-проверку
безопасности сайта), извлекает нужное подмножество (родословные + наследия, предыстории, классы, фиты,
навыки, снаряжение, состояния) и приводит к компактной схеме в `backend/data`.

- **Внутренние ID** движка — слаги из URL pf2.ru (например `/ancestries/elf` → `elf`). Контент русский «из
  коробки»; движок оперирует слагами-ID, отображение — русские строки.
- **Бережный обход:** разовый прогон, rate-limit, локальное кэширование. Результат коммитим как наш канон,
  фиксируя **дату/версию снимка** pf2.ru. Сырые страницы не коммитим как «канон».
- **Атрибуция** pf2.ru + Paizo/ORC — в README.

## 8. Игровой мир и цикл (Веха 2)

- Мир — граф **локаций** с выходами, предметами, NPC. Старт — демо-приключение (3–5 локаций + один бой).
- Поток: игрок шлёт команду (`идти север`, `осмотреть`, `атаковать гоблина`, `инвентарь`) →
  сервер парсит → движок применяет → сервер шлёт события по WebSocket → терминал печатает.
- **Парсер команд:** глагол + цель, с алиасами (`с` / `север` / `n`). Кнопки на клиенте отправляют
  те же команды.

## 9. Клиент-терминал

- Прокручиваемый лог событий; строка ввода с историей команд (стрелки вверх/вниз); панель контекстных
  кнопок (направления, частые действия, действия в бою).
- **Конструктор персонажа** — пошаговый мастер в стиле терминала (выбор из пронумерованных списков), на shadcn;
  итоговый лист персонажа с экспортом JSON.
- Стилизация CRT 80-х: моноширинный шрифт, фосфор/янтарь на чёрном, опционально скан-линии/мерцание.

## 10. Сохранения и аутентификация

- PostgreSQL (база `rpg_ero`): таблицы `users`, `characters`, `game_states` (состояние — JSONB).
- **Версионирование сейвов:** поле `schema_version` в состоянии; на раннем этапе несовместимые состояния
  инвалидируются (без миграции), позже — полноценная миграция.
- **ГПСЧ:** seed хранится в состоянии игры (воспроизводимость/реплей).
- Аутентификация по образцу home-page: JWT (python-jose) + bcrypt. На старте — простой логин,
  расширяемый позже.
- **Гейт 18+** на входе через логин/пароль; пока единственный пользователь — владелец. Проверку возраста
  для внешних пользователей добавим позже.

## 11. Взрослый контент (18+)

- Только вымышленные взрослые персонажи.
- Сцены/описания 18+ — отдельный слой данных (`content`), наполняемый пользователем; система предоставляет
  лишь триггеры и шаблоны. Боевой движок остаётся чистым и нейтральным.

## 12. Тестирование

- **TDD для движка:** каждая механика (степени успеха, MAP, инициатива, **создание персонажа: повышения,
  владения, HP, пререквизиты фитов**) покрыта юнит-тестами с детерминированным броском (инъекция ГПСЧ) —
  воспроизводимость. pytest-asyncio + ruff.
- **Парсер данных:** тесты на зафиксированных HTML-снимках pf2.ru (без обращения к сети в тестах).
- Фронт: vitest + Testing Library для ключевых компонентов терминала и конструктора персонажа.

## 13. Вехи

**Веха 1 — конструктор персонажа (самодостаточный):**
1. Конвейер данных pf2.ru → схема в `backend/data` (родословные/наследия, предыстории, классы, фиты, навыки, снаряжение).
2. Движок создания персонажа 1-го уровня по книге (`engine/chargen`) с юнит-тестами.
3. API создания/валидации персонажа + сохранение в `characters`.
4. UI-мастер конструктора в стиле терминала + лист персонажа + экспорт JSON.
5. Аутентификация (логин) и гейт 18+.

Выпускается/используется отдельно от игры (для НРИ).

**Веха 2 — игровой цикл:**
1. Мир-граф (3–5 локаций демо-приключения).
2. Парсер команд + применение действий.
3. Бой по правилам 3 действий.
4. WebSocket-поток событий, терминал-клиент.
5. Сохранение / загрузка состояния игры.

Расширение датасета, контента 18+ и мира — последующие вехи.

## 14. Лицензия / юридическое

Некоммерческий проект. При коммерциализации необходимые лицензии добываются отдельно. Атрибуция pf2.ru +
Paizo/ORC — в README.

## 15. Вне охвата v1 (YAGNI)

- Мультиплеер / общий мир (заложен архитектурно, но не реализуется).
- Уровни выше 1-го (level-up).
- Полный SRD (магия, все фиты/состояния/предметы) — только после ядра.
- Сложная экономика, крафт, фракции.
- Внешние пользователи и проверка возраста (пока единственный пользователь — владелец).
